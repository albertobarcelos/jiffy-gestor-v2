# Camada de Apresentação - Comece por Aqui

## 1. O que é a camada de apresentação

A camada de apresentação é a porta de entrada do sistema.

Ela recebe requisições externas, adapta o formato de entrada para o formato interno da aplicação e devolve respostas para o cliente.

## 2. Para que ela serve

Essa camada serve para:

1. expor interfaces de comunicação do sistema;
2. receber e encaminhar dados para a camada de aplicação;
3. transformar resultados da aplicação em resposta de protocolo (HTTP, gRPC, etc.);
4. manter separado o contrato externo da regra interna.

## 3. Como ela é usada no Jiffy

No Jiffy, a camada de apresentação é composta principalmente por:

- `controllers`;
- `routes` HTTP (Express);
- componentes de entrada por módulo (`src/modules/*/presentation`).

Também pode evoluir para incluir endpoints/handlers em outros protocolos, como **gRPC**, sem alterar a regra de negócio do domínio.

## 4. Papel arquitetural da camada

- a apresentação não contém regra central de domínio;
- a apresentação conversa com a aplicação, não com infraestrutura de forma direta para decidir negócio;
- a apresentação organiza como o sistema “fala com o mundo externo”.

## 5. Checklist rápido

1. esta camada está apenas adaptando entrada/saída?
2. há regra de negócio vazando para controller ou rota?
3. o protocolo externo está desacoplado da regra interna?
4. a camada de aplicação continua sendo o ponto de orquestração?
