# Routes na Camada de Apresentação

## 1. O que são rotas

`Routes` definem os endpoints expostos pela API e conectam método/URL ao controller responsável.

Na prática, as rotas declaram “como o cliente chama” cada funcionalidade.

## 2. Para que servem

As rotas servem para:

1. organizar entradas HTTP por módulo e contexto;
2. aplicar middlewares (auth, autorização, rate limit, validações de borda);
3. encaminhar a requisição para o método correto do controller;
4. padronizar convenções de endpoint da API.

## 3. Como rotas são usadas no Jiffy

No Jiffy, o padrão atual é:

- uso de `Router` do Express;
- arquivos de rotas por módulo/contexto (`.../presentation/...-routes.ts`);
- handlers que instanciam controller via factory (`make...Controller(req)`).

## 4. Regras de organização

### 4.1 Agrupamento por contexto

- rotas devem ser agrupadas por módulo funcional;
- evitar arquivo único gigante com endpoints de múltiplos contextos.

### 4.2 Responsabilidade da rota

- rota configura caminho, método HTTP e middlewares;
- rota não implementa regra de negócio.

### 4.3 Convenções de nomenclatura

- nomes previsíveis e consistentes;
- preferir padrões REST quando aplicável (`POST`, `GET`, `PATCH`, `DELETE`).

## 5. Exemplo correto

```ts
export const taxaRoutes = Router();

taxaRoutes.post("/", async (req, res, next) => {
  try {
    await makeTaxaController(req).create(req, res, next);
  } catch (err) {
    next(err);
  }
});
```

## 6. Exemplo incorreto

```ts
const router = Router();

router.post("/clientes", async (req, res) => {
  const cliente = Cliente.create(req.body); // regra de domínio na rota
  const created = await prisma.cliente.create({ data: cliente }); // acesso direto ao banco
  res.json(created);
});
```

Problemas:

- rota virou lugar de regra e persistência;
- quebra separação entre apresentação, aplicação e infraestrutura;
- dificulta padronização de middlewares e tratamento de erros.

## 7. Checklist rápido para rotas

1. a rota apenas configura endpoint + middleware + controller?
2. existe regra de negócio ou acesso direto ao banco na rota?
3. agrupamento por módulo está claro?
4. convenções de método/caminho estão consistentes?
