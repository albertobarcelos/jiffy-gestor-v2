# Controller Validation

## 1. O que é validação de controller

Validação de controller é a validação de **borda** da requisição.

Ela garante que a entrada recebida no protocolo externo (HTTP/gRPC) tem formato válido antes de chegar à aplicação.

## 2. Para que serve

No Jiffy, essa validação serve para:

1. validar tipos primitivos (`string`, `number`, `boolean`, arrays, datas);
2. validar formato básico (campo obrigatório, e-mail, UUID, tamanho mínimo/máximo);
3. rejeitar payload inválido cedo, com erro consistente;
4. evitar que use cases recebam dados malformados.

## 3. O que validar no controller

### 3.1 Entrada de request

- `params`
- `query`
- `body`
- headers específicos quando necessário

### 3.2 Regras de formato (não de negócio)

- presença de campos obrigatórios;
- tipo primitivo correto;
- formato sintático (ex.: e-mail, id, paginação);
- normalizações simples de borda quando aplicável.

## 4. O que não validar no controller

- invariantes de domínio;
- políticas de negócio;
- autorização complexa de fluxo;
- validações que dependem de estado profundo de negócio.

Esses pontos pertencem à aplicação e ao domínio.

## 5. Padrão adotado no Jiffy

### 5.1 Fonte do validador

Sempre que possível, o controller deve importar validador da camada de aplicação, por exemplo:

- `application/validators`
- validadores compartilhados como `PaginationValidator`

### 5.2 Fluxo

1. controller valida `req` com schema;
2. extrai input validado;
3. chama use case com input já tipado/normalizado.

## 6. Exemplo correto

```ts
import { createApiTokenInputValidator } from "modules/authentication/application/validators/validators.ts";

export class ApiKeyController {
  constructor(private readonly createApiTokenUseCase: CreateApiTokenUseCase) {}

  async create(req: Request, res: Response, next: NextFunction) {
    try {
      const input = createApiTokenInputValidator.parse(req.body);
      const dto = await this.createApiTokenUseCase.execute(input);
      return res.status(201).json(dto);
    } catch (err) {
      return next(err);
    }
  }
}
```

## 7. Exemplo incorreto

```ts
export class ApiKeyController {
  async create(req: Request, res: Response) {
    // sem validação de entrada
    const dto = await this.createApiTokenUseCase.execute(req.body);
    return res.status(201).json(dto);
  }
}
```

Problemas:

- dados inválidos chegam à aplicação;
- erros aparecem tardiamente e de forma inconsistente;
- menor previsibilidade de contrato da API.

## 8. Checklist rápido

1. controller valida `params/query/body` antes de chamar use case?
2. validação é apenas de formato e tipos primitivos?
3. validador vem da camada de aplicação (ou shared validator oficial)?
4. use case recebe input já validado?
5. regra de negócio ficou fora do controller?
