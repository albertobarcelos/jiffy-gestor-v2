# DTOs na Camada de Aplicação

## 1. O que é um DTO

`DTO` (`Data Transfer Object`) é um tipo usado para transportar dados entre camadas, com formato explícito e previsível.

Ele representa o **contrato de dados** que a aplicação expõe para fora (principalmente para a camada de apresentação).

## 2. Como DTO é usado no Jiffy

No Jiffy, DTO é utilizado principalmente para **dados de saída**.

Ou seja:

1. o domínio trabalha com entidades/value objects;
2. a aplicação transforma isso em um DTO;
3. o controller responde usando esse DTO.

## 3. Padrão de tipagem com Zod no Jiffy

### 3.1 Inferência de tipo

Sempre que possível, o DTO é criado a partir de schema Zod com `zod.infer`.

Exemplo:

```ts
import z from "zod";

export const clienteOutputValidator = z.object({
  id: z.string(),
  nome: z.string(),
  email: z.string().email().nullable(),
});

export type ClienteDTO = z.infer<typeof clienteOutputValidator>;
```

### 3.2 Vantagem desse padrão

Quando o schema muda, o tipo muda automaticamente.

Isso reduz divergência entre validação e tipagem e evita manutenção duplicada.

## 4. Fluxo recomendado

1. definir schema do DTO em `application/validators` (ou local equivalente);
2. inferir o tipo com `z.infer<typeof schema>`;
3. mapear entidade para DTO (normalmente via `mapper`);
4. retornar o DTO no use case/controller.

Observação: no Jiffy, quem geralmente traduz entidade (ou outro formato interno) para DTO é um `mapper`.

## 5. Exemplo correto

```ts
// schema + tipo
export const produtoOutputValidator = z.object({
  id: z.string(),
  nome: z.string(),
  valor: z.number(),
});

export type ProdutoDTO = z.infer<typeof produtoOutputValidator>;

// uso no mapper/use case
const dto: ProdutoDTO = {
  id: produto.id!,
  nome: produto.nome,
  valor: produto.valor.toNumber(),
};
```

## 6. Exemplo incorreto

```ts
type ProdutoDTO = {
  id: string;
  nome: string;
  valor: number;
};

// em outro arquivo, validação diferente do tipo:
const produtoOutputValidator = z.object({
  id: z.string(),
  nome: z.string().min(3),
  valor: z.string(),
});
```

Problemas:

- tipo e validação podem divergir;
- manutenção duplicada;
- maior risco de quebrar contrato sem perceber.

## 7. Checklist rápido para DTOs

1. DTO está sendo usado como contrato de saída?
2. tipo foi inferido com `z.infer` sempre que possível?
3. schema e tipo estão no mesmo contexto para evitar divergência?
4. mapeamento entidade -> DTO está explícito e previsível?
5. o domínio continua sem depender de DTO?
