# Value Objects no Domínio

## 1. O que é um value object

`Value Object` é um objeto de domínio definido pelo **valor**, não por identidade.

Se dois objetos têm os mesmos valores, eles representam a mesma coisa no negócio.

Exemplos comuns:

- dinheiro (`Money`);
- e-mail;
- CPF/CNPJ;
- Desconto;
- Username;
- Endereço;

## 2. Para que é usado

No domínio, value objects são usados para:

1. garantir validação forte em tipos críticos;
2. evitar primitivos soltos (`string`, `number`) espalhados por toda regra;
3. encapsular comportamento do conceito (ex.: somar valores monetários);
4. aumentar legibilidade e reduzir estados inválidos.

## 3. Características esperadas

### 3.1 Imutabilidade

- após criado, o value object não deve mudar de estado;
- operações devem retornar novo objeto (não mutar o atual).

### 3.2 Validação no ponto de criação

- `create()` valida regras de negócio do conceito;
- se inválido, deve lançar erro de domínio.

### 3.3 Reconstituição

- `fromDatabase()` (ou equivalente) reidrata sem repetir validações de entrada;
- usar somente quando os dados já são confiáveis por origem.

### 3.4 Sem responsabilidade de infraestrutura

- não acessar banco, API, fila, cache ou framework;
- foco em regra e comportamento do conceito.

## 4. Quando criar um value object

Crie um value object quando:

1. há regra de validação recorrente;
2. há comportamento de negócio associado ao tipo;
3. o conceito aparece em muitos pontos do domínio;
4. usar primitivo deixa o código ambíguo ou frágil.

## 5. Exemplo correto

Exemplo alinhado ao projeto (`Money`):

```ts
export class Money {
    private constructor(private readonly _value: number) {}

    static create(value: number): Money {
        if (value < 0) throw new Error("Valor não pode ser negativo");
        return new Money(Number(value.toFixed(2)));
    }

    static fromDatabase(value: number): Money {
        return new Money(value);
    }

    sum(other: Money): Money {
        return new Money(this._value + other._value);
    }

    toNumber(): number {
        return this._value;
    }
}
```

Pontos positivos:

- criação controlada;
- regra de validação centralizada;
- comportamento do conceito encapsulado;
- operação retornando novo valor.

## 6. Exemplo incorreto

```ts
type Produto = {
    preco: number;
};

function aplicarDesconto(preco: number, desconto: number): number {
    return preco - desconto;
}
```

Problemas:

- regra monetária espalhada;
- tipos primitivos sem proteção de domínio;
- nenhuma validação (valor negativo, arredondamento, etc.);
- baixa expressividade semântica.

## 7. Checklist rápido para value objects

1. o objeto é definido por valor e não por identidade?
2. existe `create()` com validação de negócio?
3. é imutável?
4. possui comportamento do conceito (não só getters)?
5. removeu primitivos frágeis do domínio?
