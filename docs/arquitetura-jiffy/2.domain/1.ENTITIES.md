# Entidades no Domínio

## 1. O que é uma entidade

Uma entidade é um objeto de domínio com **identidade própria** e ciclo de vida.

Mesmo que seus atributos mudem, ela continua sendo a mesma entidade enquanto sua identidade for a mesma (ex.: `id`).

Exemplos comuns no Jiffy:

- `Usuario`
- `Empresa`
- `Cliente`
- `Caixa`
- `Produto`
- `GrupoProduto`
- `Venda`

## 2. Para que é usado

Uma entidade é usada para:

1. representar conceitos centrais do negócio com identidade;
2. proteger regras e invariantes relacionadas ao seu estado;
3. expor comportamentos de negócio de forma explícita;
4. impedir estados inválidos durante criação e atualização.

## 3. Estrutura recomendada no Jiffy

### 3.1 Identidade e estado

- toda entidade deve ter identidade estável (`id` ou equivalente);
- o estado interno deve ser controlado pela própria entidade;
- alterações devem passar por métodos que validem regras de negócio.

### 3.2 Formas de instanciação

No Jiffy, a entidade normalmente possui ao menos dois caminhos:

1. `create()`: aplica validações de negócio e define defaults.
2. `fromDatabase()`: reconstitui a entidade a partir de dados já persistidos.

Regra prática:

- `create()` é caminho de entrada de novos dados.
- `fromDatabase()` é caminho de reidratação.

### 3.3 Comportamento dentro da entidade

- comportamentos que alteram estado devem estar na entidade;
- cada método deve expressar uma intenção de negócio;
- métodos devem manter as invariantes válidas após a execução.

## 4. O que não pode conter em entidades

- colocar lógica de infraestrutura (ORM, HTTP, fila, e-mail);
- usar entidade apenas como "container de dados" sem comportamento;
- permitir alteração direta de campos sem validação;
- misturar orquestração de caso de uso dentro da entidade.

## 5. Exemplo correto

```ts
class Produto {
    private constructor(private props: CreateProduto) {}

    static create(props: CreateProdutoInput): Produto {
        // define defaults
        const resolvedProps = {
            ...props,
            ativo: props.ativo ?? true,
            gruposComplementos: [],
            dataCriacao: new Date(),
            dataAtualizacao: new Date(),
        };

        // vaalida
        const validatedProps = this.validate(resolvedProps);

        // instancia
        return new Produto(validatedProps);
    }

    static fromDatabase(props: ProdutoProps): Produto {
        return new Produto(props);
    }

    update(props: UpdateProdutoInput) {
        // regra para atualização, validação, etc
    }

    desativar(): void {
        this.props.ativo = false;
        this.dataAtualizacao: new Date(),
    }

    private static validate(props: CreateProduto) {
        return createProdutoValidator.parse(props);
    }

    // Qualquer outro método que altera o estado do produto
}
```

## 6. Exemplo incorreto

```ts
class Produto {
    constructor(
        public id: string,
        public nome: string,
        public preco: number,
        public ativo: boolean
    ) {}
}

// Problemas:
// - sem validação de criação
// - estado totalmente exposto para alteração externa
// - nenhum comportamento de negócio encapsulado
```

## 7. Checklist rápido para entidades

1. a entidade tem identidade explícita?
2. regras de criação estão em `create()`?
3. existe método de reconstituição (ex.: `fromDatabase()`)?
4. invariantes são preservadas após cada operação?
5. a entidade está livre de detalhes de infraestrutura?
6. toda lógica que altera o estado da entidade está na própria entidade?
